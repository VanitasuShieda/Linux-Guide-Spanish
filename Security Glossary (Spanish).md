Una lista de términos clave de seguridad de la información para software y hardware. **Fuentes:** [NIST Federal Information Processing Standards (FIPS)](https://csrc.nist.gov/publications/fips), la [Special Publication (SP) 800 series](https://csrc.nist.gov/publications/sp), [NIST Interagency Reports (NISTIRs)](https://csrc.nist.gov/publications/nistir), y la [Committee for National Security Systems Instruction 4009 (CNSSI-4009)](https://www.cnss.gov/CNSS/issuances/Instructions.cfm).


A

Access – Acceso: Capacidad y medios para comunicarse o interactuar con un sistema, usar recursos del sistema, obtener conocimiento de la información que contiene el sistema o controlar componentes y funciones del sistema.
EN: Ability and means to communicate with or otherwise interact with a system, to use system resources to handle information, to gain knowledge of the information the system contains, or to control system components and functions.
SOURCE: CNSSI-4009

Access Authority – Autoridad de acceso: Entidad responsable de monitorear y otorgar privilegios de acceso a otras entidades autorizadas.
EN: An entity responsible for monitoring and granting access privileges for other authorized entities.
SOURCE: CNSSI-4009

Access Control – Control de acceso: Proceso de permitir o negar solicitudes específicas para: 1) obtener y usar información y servicios de procesamiento relacionados; y 2) entrar a instalaciones físicas específicas (p. ej., edificios federales, establecimientos militares, entradas de pasos fronterizos).
EN: The process of granting or denying specific requests to: 1) obtain and use information and related information processing services; and 2) enter specific physical facilities (e.g., federal buildings, military establishments, border crossing entrances).
SOURCE: FIPS 201; CNSSI-4009

Access Control List (ACL) – Lista de control de acceso: 1) Una lista de permisos asociados con un objeto que especifica quién o qué puede acceder al objeto y qué operaciones se permiten; 2) Un mecanismo que implementa control de acceso enumerando las entidades del sistema permitidas y declarando los modos de acceso otorgados a cada entidad.
EN: 1. A list of permissions associated with an object. The list specifies who or what is allowed to access the object and what operations are allowed to be performed on the object.
2. A mechanism that implements access control for a system resource by enumerating the system entities that are permitted to access the resource and stating, either implicitly or explicitly, the access modes granted to each entity.
SOURCE: CNSSI-4009

Access Control Lists (ACLs) – Listas de control de acceso: Registro de usuarios (incluyendo grupos, máquinas, procesos) que tienen permiso para usar un recurso del sistema, y los tipos de acceso que se les han permitido.
EN: A register of: 1. users (including groups, machines, processes) who have been given permission to use a particular system resource, and 2. the types of access they have been permitted.
SOURCE: SP 800-12

Access Control Mechanism – Mecanismo de control de acceso: Medidas de seguridad (hardware y software, controles físicos, procedimientos operativos y de gestión) diseñadas para detectar y denegar accesos no autorizados y permitir accesos autorizados a un sistema de información.
EN: Security safeguards (i.e., hardware and software features, physical controls, operating procedures, management procedures, and various combinations of these) designed to detect and deny unauthorized access and permit authorized access to an information system.
SOURCE: CNSSI-4009

Access Level – Nivel de acceso: Categoría dentro de una clasificación de seguridad que limita la entrada o la conectividad del sistema sólo a personas autorizadas.
EN: A category within a given security classification limiting entry or system connectivity to only authorized persons.
SOURCE: CNSSI-4009

Access List – Lista de acceso: Registro de individuos autorizados para admisión a un área controlada.
EN: Roster of individuals authorized admittance to a controlled area.
SOURCE: CNSSI-4009

Access Point – Punto de acceso: Dispositivo que conecta lógicamente dispositivos cliente inalámbricos operando en infraestructura entre sí y proporciona acceso a un sistema de distribución, normalmente la red cableada de la organización.
EN: A device that logically connects wireless client devices operating in infrastructure to one another and provides access to a distribution system, if connected, which is typically an organization’s enterprise wired network.
SOURCE: SP 800-48; SP 800-121

Access Profile – Perfil de acceso: Asociación de un usuario con una lista de objetos protegidos a los que el usuario puede acceder.
EN: Association of a user with a list of protected objects the user may access.
SOURCE: CNSSI-4009

Access Type – Tipo de acceso: Privilegio para realizar una acción sobre un objeto (leer, escribir, ejecutar, anexar, modificar, borrar, crear).
EN: Privilege to perform action on an object. Read, write, execute, append, modify, delete, and create are examples of access types.
SOURCE: CNSSI-4009

Activation Data – Datos de activación: Datos privados, distintos de las claves, que se requieren para acceder a módulos criptográficos.
EN: Private data, other than keys, that are required to access cryptographic modules.
SOURCE: SP 800-32

Active Attack – Ataque activo: Un ataque que altera un sistema o datos.
EN: An attack that alters a system or data.
SOURCE: CNSSI-4009

Active Content – Contenido activo: Software en varias formas capaz de ejecutar o desencadenar acciones en una plataforma sin intervención del usuario.
EN: Software in various forms that is able to automatically carry out or trigger actions on a computer platform without the intervention of a user.
SOURCE: CNSSI-4009

EN: Security testing that involves direct interaction with a target, such as sending packets to a target.
SOURCE: SP 800-115

Advanced Encryption Standard – Estándar de Encriptación Avanzado (AES): Especifica un algoritmo criptográfico aprobado por el gobierno de EE. UU. para proteger datos electrónicos; es un cifrador de bloque simétrico que procesa bloques de 128 bits y admite claves de 128, 192 y 256 bits.
EN: The Advanced Encryption Standard specifies a U.S. government-approved cryptographic algorithm that can be used to protect electronic data. The AES algorithm is a symmetric block cipher that can encrypt (encipher) and decrypt (decipher) information... 
SOURCE: FIPS 197

B

Blacklisting – Lista negra: Proceso por el cual el sistema invalida un ID de usuario por acciones inapropiadas. Un ID en lista negra no puede usarse para iniciar sesión, incluso con el autenticador correcto. La inclusión y la eliminación de la lista negra son eventos relevantes de seguridad. También aplica a bloques contra direcciones IP.
EN: The process of the system invalidating a user ID based on the user’s inappropriate actions. A blacklisted user ID cannot be used to log on to the system, even with the correct authenticator. Blacklisting and lifting of a blacklisting are both security-relevant events. Blacklisting also applies to blocks placed against IP addresses to prevent inappropriate or unauthorized use of Internet resources.
SOURCE: CNSSI-4009

Blue Team – Equipo azul: Grupo responsable de defender el uso de sistemas de información de una empresa manteniendo su postura de seguridad frente a un grupo de atacantes simulados (Red Team). También puede realizar evaluaciones operativas de vulnerabilidades y recomendar mitigaciones.
EN: The group responsible for defending an enterprise’s use of information systems by maintaining its security posture against a group of mock attackers (i.e., the Red Team)...
SOURCE: CNSSI-4009

Cloud Computing – Computación en la nube: Modelo que permite acceso ubicuo, conveniente y bajo demanda a un conjunto compartido de recursos computacionales configurables que pueden aprovisionarse y liberarse rápidamente con mínimo esfuerzo de gestión.
